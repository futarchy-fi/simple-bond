# temporal-fleet-2nm-r1 Analysis: document `SimpleBondV4` getters and missing function NatSpec

## Summary

This child task is a narrowed documentation pass on [`contracts/SimpleBondV4.sol`](/tmp/temporal-worktrees/task-temporal-fleet-2nm-r1-document-simplebondv4-functions-and-getters/contracts/SimpleBondV4.sol#L1).

The contract already has broad NatSpec coverage, but the remaining public read surface is still inconsistent. The autogenerated getters for [`nextBondId`](/tmp/temporal-worktrees/task-temporal-fleet-2nm-r1-document-simplebondv4-functions-and-getters/contracts/SimpleBondV4.sol#L64), [`bonds`](/tmp/temporal-worktrees/task-temporal-fleet-2nm-r1-document-simplebondv4-functions-and-getters/contracts/SimpleBondV4.sol#L65), [`challenges`](/tmp/temporal-worktrees/task-temporal-fleet-2nm-r1-document-simplebondv4-functions-and-getters/contracts/SimpleBondV4.sol#L66), and [`judges`](/tmp/temporal-worktrees/task-temporal-fleet-2nm-r1-document-simplebondv4-functions-and-getters/contracts/SimpleBondV4.sol#L67) have no NatSpec, and several explicit public/view functions are missing `@param` and `@return` tags.

This should remain a NatSpec-only change in a single Solidity file. No behavior, ABI, storage, or test logic should change.

## Scope Notes

- Generated getters must be documented on the storage declarations themselves. Solidity synthesizes those external accessors from the `public` state variables, so the NatSpec belongs above [`nextBondId`](/tmp/temporal-worktrees/task-temporal-fleet-2nm-r1-document-simplebondv4-functions-and-getters/contracts/SimpleBondV4.sol#L64), [`bonds`](/tmp/temporal-worktrees/task-temporal-fleet-2nm-r1-document-simplebondv4-functions-and-getters/contracts/SimpleBondV4.sol#L65), [`challenges`](/tmp/temporal-worktrees/task-temporal-fleet-2nm-r1-document-simplebondv4-functions-and-getters/contracts/SimpleBondV4.sol#L66), [`judges`](/tmp/temporal-worktrees/task-temporal-fleet-2nm-r1-document-simplebondv4-functions-and-getters/contracts/SimpleBondV4.sol#L67), and [`judgeMinFees`](/tmp/temporal-worktrees/task-temporal-fleet-2nm-r1-document-simplebondv4-functions-and-getters/contracts/SimpleBondV4.sol#L69), not via new wrapper functions.
- The task description is narrower than the parent `temporal-fleet-2nm` analysis. This pass should stay focused on the specified getters and functions only; event and custom-error documentation can be left untouched unless a tiny wording adjustment is required for consistency in the same hunk.
- The wording needs to reflect actual getter semantics:
  - [`challenges`](/tmp/temporal-worktrees/task-temporal-fleet-2nm-r1-document-simplebondv4-functions-and-getters/contracts/SimpleBondV4.sol#L66) is an indexed queue-entry getter, not an aggregate history summary.
  - [`judges`](/tmp/temporal-worktrees/task-temporal-fleet-2nm-r1-document-simplebondv4-functions-and-getters/contracts/SimpleBondV4.sol#L67) exposes the `JudgeInfo.registered` flag only.
  - [`judgeMinFees`](/tmp/temporal-worktrees/task-temporal-fleet-2nm-r1-document-simplebondv4-functions-and-getters/contracts/SimpleBondV4.sol#L69) is keyed by both judge and token.
